| 경로                | 설명                                                                                                                                         |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **src/**          | 애플리케이션의 모든 소스 코드가 위치하는 루트 디렉터리입니다.                                                                                                         |
| ┣ **app/**        | **앱의 핵심 구조와 진입점 역할을 하는 폴더**로, 라우팅(`AppRouter.jsx`), 레이아웃(`RootLayout.jsx`), 전역 상태 관리 등 전체 앱의 뼈대를 구성하는 파일들이 위치합니다.                          |
| ┣ **pages/**      | **페이지 단위 컴포넌트**를 저장하는 폴더입니다. 각 파일은 `react-router-dom`에서 개별 라우트로 연결됩니다. 예: `Home.jsx`, `About.jsx`, `Login.jsx` 등.                          |
| ┣ **components/** | **공통 UI 컴포넌트**를 모아두는 폴더입니다. 여러 페이지에서 재사용 가능한 버튼, 카드, 모달, 헤더, 푸터 등의 컴포넌트를 관리합니다.                                                            |
| ┣ **styles/**     | **전역 스타일과 테마 관련 파일**을 관리하는 폴더입니다. 예를 들어, `GlobalStyle.js`(전역 CSS 정의)와 `theme.js`(색상, 폰트 크기 등 테마 정의)가 포함됩니다.                                |
| ┣ **assets/**     | **정적 리소스(Static Assets)**를 저장하는 폴더입니다. 폰트(`fonts/`), 이미지(`images/`), 아이콘(`icons/`) 등과 같은 파일이 여기에 들어갑니다.                                    |
| ┣ **api/**        | **서버 통신 관련 코드**를 관리하는 폴더입니다. `axios` 인스턴스 설정(`client.js`), 각 엔드포인트별 API 함수(`userApi.js`, `postApi.js`) 등을 작성합니다.                           |
| ┣ **hooks/**      | **React 커스텀 훅(Custom Hooks)**을 정의하는 폴더입니다. 예: `useFetch.js`, `useToggle.js`, `usePagination.js` 등 — 여러 컴포넌트에서 공통으로 사용하는 상태 로직을 재사용합니다.     |
| ┣ **utils/**      | **공통 유틸리티 함수**를 관리하는 폴더입니다. 예: `formatDate.js`, `calculateRatio.js`, `validateEmail.js` 등 — 비즈니스 로직과 UI에 종속되지 않은 순수 함수들이 위치합니다.            |
| ┗ **main.jsx**    | **앱의 진입점(entry point)**입니다. `ReactDOM.createRoot()`를 통해 애플리케이션을 렌더링하며, `ThemeProvider`, `BrowserRouter`, `GlobalStyle` 등을 이곳에서 한 번만 설정합니다. |
